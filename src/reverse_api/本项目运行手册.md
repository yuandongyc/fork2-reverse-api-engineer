不，`pip install -e .` **不会**创建虚拟环境。

它会安装到**当前活动的 Python 环境**（系统 Python 或已激活的虚拟环境）。

如果要用 pip + 虚拟环境，需要手动两步：
```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 激活并安装
.\.venv\Scripts\activate
pip install -e .
```

---

**对比：**

| 操作 | uv | pip |
|------|-----|-----|
| 创建虚拟环境 | `uv sync`（自动） | `python -m venv .venv`（手动） |
| 安装依赖 | `uv sync` | `pip install -e .` |
| 运行 | `uv run reverse-api-engineer` | `.\.venv\Scripts\activate` 后运行 |

所以 uv 的优势就是 **一步到位**，不需要手动管理虚拟环境。

在 Windows 上设置环境变量有两种方式：

### 方式 1: 临时设置（当前终端有效）
```cmd
set MINIMAX_API_KEY=你的API密钥
```
这种方法只在当前命令行窗口有效，关闭后失效。

### 方式 2: 永久设置（用户级）
```cmd
setx MINIMAX_API_KEY "你的API密钥"
```
这种方法永久生效，保存到用户环境变量。

### 方式 3: 直接修改配置文件
在 `~/.reverse-api/config.json` 中添加：
```json
{
  "minimax_model": "MiniMax-M2.7",
  "sdk": "minimax"
}
```
（注意：API Key 只支持环境变量方式，因为不应该明文保存到配置文件）
